# Mailer Pipeline — Future Plan

> **Status:** Not yet built. Detailed implementation plan to be created when this becomes the active workstream.

## What This Completes

This is **step 6 (SEND)** in the legal chain — the final automation step before the bureau's obligation kicks in. Once this works, the full end-to-end process is:

```
ACQUIRE  → Pull reports from AnnualCreditReport.com
EXTRACT  → Parse every data point with 100% accuracy
ANALYZE  → Identify material inaccuracies
HIGHLIGHT → Mark evidence on original report pages
GENERATE → Build CFPB-standard dispute letter
SEND     → Mail letter + highlighted report + ID docs to bureaus  ← THIS
RESULT   → Bureau reinvestigates or fails to → cause of action
```

## What Gets Mailed

Each dispute package sent to a bureau contains:

1. **Dispute letter** (DOCX/PDF) — generated by the letter builder, CFPB-formatted
2. **Highlighted credit report** (PDF) — original report pages with yellow bounding boxes marking disputed items
3. **Identification documents** (uploaded by consumer) — typically:
   - Government-issued photo ID
   - Proof of address (utility bill, bank statement)
   - SSN verification (if required by bureau)

These three components form a single certified mail package per bureau.

## Likely Approach

- **Mail API:** Lob.com or similar print-and-mail API that handles physical letter delivery with tracking
- **Certified mail:** Disputes should be sent via certified mail with return receipt — this creates the legal proof of delivery that starts the bureau's 30-day reinvestigation clock under FCRA § 611
- **Tracking integration:** Certified mail tracking numbers feed back into the application so the consumer/attorney can monitor delivery and response deadlines
- **Per-bureau packaging:** Each bureau gets its own package (Equifax, Experian, TransUnion have different mailing addresses)
